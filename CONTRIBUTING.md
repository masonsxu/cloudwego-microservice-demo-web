# 贡献指南

感谢你考虑为 Cloudwego微服务管理平台 做出贡献！

## 如何贡献

### 报告Bug

如果你发现了bug，请：
1. 在 [Issues](https://github.com/masonsxu/cloudwego-microservice-demo-web/issues) 中搜索，确认是否已经有人报告过
2. 如果没有，创建一个新的Issue，使用Bug报告模板
3. 提供尽可能详细的信息，包括复现步骤、环境信息和截图

### 提出新功能

如果你有新功能的想法：
1. 在 [Issues](https://github.com/masonsxu/cloudwego-microservice-demo-web/issues) 中搜索，确认是否已经有人提出过
2. 如果没有，创建一个新的Issue，使用功能请求模板
3. 详细描述你的想法，以及为什么你认为这个功能很重要

### 提交代码

1. Fork 本仓库
2. 创建你的特性分支 (`git checkout -b feature/AmazingFeature`)
3. 提交你的更改 (`git commit -m 'Add some AmazingFeature'`)
4. 推送到分支 (`git push origin feature/AmazingFeature`)
5. 开启一个 Pull Request

## 开发规范

### 代码风格

- 使用 TypeScript 进行类型检查
- 遵循 Vue 3 Composition API 最佳实践
- 使用 Prettier 格式化代码
- 使用 ESLint 进行代码检查

### 提交信息

遵循 [Conventional Commits](https://www.conventionalcommits.org/) 规范：

```
feat: 添加新功能
fix: 修复bug
docs: 文档更新
style: 代码格式调整
refactor: 代码重构
test: 测试相关
chore: 构建/工具链相关
```

### Pull Request

- 确保你的代码通过所有测试
- 更新相关文档
- 使用 PR 模板描述你的更改
- 等待代码审查

## 许可证

通过向本项目贡献代码，你同意你的代码将在 MIT 许可证下发布。
